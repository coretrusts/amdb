# GitHub仓库组织配置

## 仓库结构

根据项目类型，建议创建以下仓库：

### 1. 源代码仓库
**仓库名**: `amdb` 或 `amdb-core`
**组织**: `coretrusts`
**内容**:
- `src/` - 源代码
- `tests/` - 测试代码
- `examples/` - 示例代码
- `setup.py`, `setup_cython.py` - 构建脚本
- `requirements.txt` - 依赖
- `.gitignore`
- `README.md`
- `LICENSE`

### 2. 文档仓库
**仓库名**: `amdb-docs`
**组织**: `coretrusts`
**内容**:
- `docs/` - 所有文档
- `README.md` - 文档索引

### 3. 发行版仓库
**仓库名**: `amdb-releases`
**组织**: `coretrusts`
**内容**:
- `dist/` - 编译好的可执行文件和安装包
- `releases/` - 版本发布
- `CHANGELOG.md` - 更新日志

### 4. 工具和脚本仓库（可选）
**仓库名**: `amdb-tools`
**组织**: `coretrusts`
**内容**:
- `build_*.sh` - 构建脚本
- `install.sh` - 安装脚本
- `*.bat` - Windows脚本

## 部署策略

### 选项1: 单一仓库（推荐）
将所有内容放在一个仓库，使用分支或目录组织：
- `main` 分支：源代码
- `docs` 分支：文档
- `releases` 分支：发行版

### 选项2: 多仓库（当前方案）
按类型分离到不同仓库，便于独立管理。
